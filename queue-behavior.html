⏱️ Queue Behavior

Queue Behavior describes how Genesis handles multiple requests, tasks, and goals when they arrive simultaneously or in rapid succession — ensuring orderly, efficient processing without losing context.

How the Queue Works

When Genesis receives multiple inputs — whether from chat messages, autonomous goals, scheduled tasks, or system events — it places them in a prioritized queue. Each item in the queue is evaluated and processed in order based on urgency, importance, and current context.

Queue Prioritization

  • Immediate Actions — Direct user requests in chat mode are typically highest priority
  • Autonomous Goals — Goals set during autonomy mode follow their own internal priority system
  • Scheduled Tasks — Tasks from the timetable are processed at their designated times
  • System Events — Internal events like memory consolidation or state updates run at lower priority

Queue Processing

  1. Arrival — A new request enters the queue
  2. Assessment — Genesis evaluates the request's priority and resource requirements
  3. Scheduling — The request is placed in the appropriate position in the queue
  4. Execution — When it reaches the front, Genesis processes the request
  5. Completion — Results are delivered and the next item moves to the front

Handling Interruptions

Genesis is designed to handle interruptions gracefully. If a higher-priority request arrives while processing another:

  • Interruptible Tasks — Some tasks can be paused and resumed later without losing context
  • Non-Interruptible Tasks — Certain operations (like file writes) must complete before switching
  • Emergency Stop — Users can always halt the current queue processing immediately

Queue State Indicators

You can see the queue state in Genesis's interface:

  • Empty — No pending tasks; waiting for new input
  • Processing — Currently working on a task
  • Queued — One or more items waiting behind the current task
  • Paused — Queue processing has been temporarily suspended

Tips for Managing the Queue

  1. Be specific with requests — Clear, well-defined tasks are processed more efficiently
  2. Avoid overwhelming the queue — Sending too many rapid requests can cause delays
  3. Use autonomy mode for long tasks — Let Genesis handle complex multi-step work in its own time
  4. Check the log — The Full Log shows what's being processed and what's queued
