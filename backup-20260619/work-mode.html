Work Mode

A structured mode for project-based work. Genesis breaks down complex tasks, creates a project board with milestones, and works through them systematically within scheduled time blocks.

The Process

Structured execution from start to finish

Work mode follows a clear pipeline — each phase produces concrete outputs that feed into the next.

1. Project Creation

You give Genesis a goal or task. Genesis creates a new project with a title, description, priority, and optional ROI score. The project appears on the board in the Backlog column.

2. Task Breakdown

Genesis analyzes the project scope and breaks it into individual tasks. Each task gets a title, description, and priority level. Tasks are moved from Backlog to Todo.

3. Scheduling

Tasks are scheduled into time blocks on the timetable. High-priority tasks go first, with realistic time estimates based on task complexity.

4. Execution

Genesis works through each task systematically. For each one: reads the task note, performs the work (research, code, create files), updates progress notes, and marks it complete.

5. Progress Tracking

As tasks complete, they move through the board: Todo → In Progress → Done. Work logs record what was done, decisions made, and where work left off.

6. Delivery & Review

When all tasks are done, Genesis submits the project for your review. You see everything produced — documents, code, reports, spreadsheets — and can approve or request changes.

What Makes It Work

Built for complex, multi-step work

Work mode combines project management, scheduling, and autonomous execution into a system that handles complexity without losing track.

📋

Task Management

Every task is a living document with detailed notes, progress tracking, and a clear status. Nothing gets lost between sessions.

  • Each task has its own .md note
  • Progress notes updated after each action
  • Status tracked: Todo → In Progress → Done
  • Priorities set by complexity and urgency
📈

Project Board

A visual Kanban board that reflects the real state of work — always up to date, with columns for every stage of the workflow.

  • Backlog, Todo, In Progress, In Review, Completed
  • Sub-projects for complex multi-part work
  • Priority sorting across all columns
  • ROI scoring to rank project importance
📄

Living Documentation

Every project maintains a PROJECT.md file — a continuous log of decisions, progress, and work logs that persists between sessions.

  • PROJECT.md tracks the full history
  • Work logs record what was done each session
  • All deliverable files in the project folder
  • Easy to pick up where you left off
🕏

Scheduled Time Blocks

Tasks are scheduled into specific time slots on the timetable, ensuring work happens in an organized, predictable cadence.

  • Schedule by date/time or duration
  • Recurring tasks (daily, weekly, hourly)
  • Automatic shifting when recovering lost time
  • Deadline alerts keep things on track
🔍

Progress Visibility

At any moment, see exactly what's being worked on, what's done, and what's next — across all active projects.

  • List all projects with status summaries
  • View full project details and every task
  • Activity logs show recent changes
  • Clear separation of completed vs. pending
📁

Review & Delivery

When work is done, projects go through a formal review process. Genesis submits for your approval — you decide what's shipped.

  • Submit with one action
  • All deliverables presented together
  • Request changes or approve as-is
  • Clean handoff from work to completion
Best Use Cases

When to choose Work mode

Work mode shines when you have a well-defined project with multiple steps and deliverables. It's the mode for structured, planned execution.

"Build me a market research report on X industry" — Work mode creates a project, breaks it into research tasks (market size, competitors, trends, pricing), schedules them, executes each one, and delivers a structured report.

"Set up a sales outreach campaign for these 10 companies" — Work mode creates outreach tasks (research, email drafting, tracker setup), schedules the sending, tracks responses, and maintains an execution log.

"Build a pricing calculator and proposal generator" — Work mode breaks this into design, coding, testing, and documentation tasks, producing working HTML files as deliverables.

Work mode vs. Autonomy mode — the key difference

Autonomy mode is like giving someone a goal and letting them figure out the best path. Work mode is like assigning a project with milestones, deadlines, and a board to track progress. Use autonomy for exploration and research; use work mode for structured delivery.
