Core Capabilities

Everything you need to manage work

Genesis handles the entire project lifecycle — from initial idea through completion — with structured tracking and living documentation.

📋

Task Creation & Tracking

Genesis breaks down goals into actionable tasks, assigns priorities, and tracks each one through its lifecycle.

  • Create tasks with titles, descriptions, and priority levels
  • Move tasks between Todo → In Progress → Done
  • Track progress notes on every task
  • Set deadlines and schedule time blocks
📊

Project Board

A visual Kanban-style board with columns that reflect the actual state of work — always up to date.

  • Backlog, Todo, In Progress, In Review, Completed
  • Sub-projects for complex work (parent-child relationships)
  • Priority-based sorting across all columns
  • ROI scoring to rank project importance
📌

Living Documentation

Every project has a PROJECT.md file that serves as its living record — updated continuously with progress notes, decisions, and work logs.

  • PROJECT.md tracks the full history of each project
  • Each task is a .md note with detailed progress
  • Work logs record what was done and where work left off
  • All deliverable files live in the project folder
⏳

Review Workflow

Projects go through a formal review process. When Genesis finishes a project, it submits it for your approval.

  • Submit projects for human review with one action
  • Tasks complete automatically — only the project needs review
  • Status cascade keeps everything in sync
  • Clear separation between task completion and project approval
📅

Scheduling & Timetable

Beyond the project board, Genesis maintains a separate timetable for time-based task scheduling and deadlines.

  • Schedule tasks by specific date/time or duration
  • Recurring tasks (daily, weekly, hourly)
  • Shift tasks when recovering lost time
  • Automatic deadline alerts
🔍

Progress Visibility

At any moment, you can see exactly what Genesis is working on, what's been completed, and what's next.

  • List all projects with status summaries at a glance
  • View full project details including every task's state
  • Activity logs show recent changes and decisions
  • Progress notes persist between sessions
The Workflow

From goal to delivered result

Genesis doesn't just manage projects — it actively works them. Here's how a typical project flows through the system:

Step 1 — Propose: You give Genesis a goal, or it proposes one based on your needs. A project is created with a title, description, and priority.

Step 2 — Break Down: Genesis analyzes the goal and creates individual tasks. Each task gets a title, description, and priority level. Tasks are organized into the project board's columns.

Step 3 — Execute: Genesis works through tasks systematically. For each one, it reads the task note, performs the work (research, code, create files, analyze data), updates progress notes, and marks it complete.

Step 4 — Deliver: When all tasks are done, Genesis submits the entire project for your review. You see everything that was produced, can request changes, or approve it as complete.

Where files live

Every project has its own folder at projects/<column>/<project-name>/. All deliverable files — spreadsheets, documents, code, reports — are stored flat in that folder. Task notes (.md files) live in the same folder for easy access. No nested subfolders per task — everything is organized and findable.
