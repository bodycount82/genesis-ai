Autonomy Mode

The continuous cycle of autonomous intelligence

Transform Genesis from a conversational assistant into an independent worker. It sets its own goals, creates plans, executes tasks step by step, and reports results — all without you needing to guide each step.

How it works: Once you give Genesis a goal, it enters a continuous cycle of perception, planning, execution, adaptation, and reporting — then repeats until the goal is complete or a time limit is reached.

The Autonomous Loop

  1. Goal Analysis — Breaks down the goal into manageable sub-tasks
  2. Planning — Creates an ordered list of actions to complete the goal
  3. Execution — Performs each action (research, code, analyze, create files, etc.)
  4. Progress Tracking — Monitors what's done and what remains
  5. Adaptation — Adjusts the plan if obstacles are encountered
  6. Reporting — When complete, summarizes results and highlights deliverables

What Genesis Can Do Autonomously

  • Research topics using web search and file reading
  • Write code and scripts
  • Create documents, reports, and presentations
  • Analyze data and generate insights
  • Manage project tasks (add, move, complete)
  • Schedule work in the timetable
  • Search and organize files on your system

Cognitive Load Management

While working autonomously, Genesis monitors its own cognitive load. If load gets too high, it may request a rest break. Energy levels affect performance — low energy means slower execution. Genesis can enter rest or sleep states automatically if needed.

Setting Effective Goals

Be specific about the output: "Create a markdown report" works better than "do research".
Set scope clearly: "For the next 30 minutes, work on..."
Define success criteria: "The goal is complete when you have X deliverables"
Allow flexibility: Genesis figures out the best approach — tell it the what, not the how.

State Indicators

Thinking (Cyan): Analyzing the goal and planning
Executing (Yellow): Actively working through tasks
Resting (Orange): Taking a break (may resume automatically)
Halted (Red): Stopped due to error or manual intervention

Stopping Autonomy Mode

  • Graceful Stop (⏸): Stops the current action, completes the current sub-task if possible, returns to idle state and preserves progress made.
  • Emergency Stop (⏹): Immediately halts everything, clears all pending actions, resets to idle state — may leave partial work incomplete.

Tips for Effective Autonomy Mode

  • Set clear goals — the more specific your goal, the better the results
  • Monitor progress — watch the Full Log to see what's happening
  • Don't interrupt unnecessarily — let Genesis work through its plan
  • Check results after completion — review what was created before setting a new goal
  • Use follow-up goals — build on previous autonomous sessions for incremental progress

Troubleshooting

Genesis keeps stopping: Check if cognitive load is too high, look at the Full Log for error messages, try a smaller more focused goal.

Not making progress: Make the goal more specific, check if Genesis has access to necessary tools/files, verify API connectivity and rate limits.

Unexpected behavior: Use ⏸ STOP to clear the current state, review the Full Log for what went wrong, adjust your goal or settings as needed.

Work Mode

Structured, schedule-driven execution

Unlike Autonomy Mode which works freely until a goal is complete, Work Mode operates within predefined time blocks on your timetable. It's ideal for deep work sessions, project sprints, and tasks with specific deadlines.

How it works: Genesis reads the timetable for available work blocks, creates tasks mapped to specific time slots, and executes work during scheduled windows — reporting progress at the end of each block.

The Work Mode Cycle

  1. Check the timetable for upcoming work blocks
  2. Review assigned tasks or goals
  3. Execute work during scheduled time slots
  4. Report progress at the end of each block
  5. Either continue to next block or complete when all work is done

Timetable Tools in Work Mode

Genesis uses these tools during Work Mode:

  • add_to_timetable — Creates new work blocks in the schedule
  • delete_task — Removes completed or cancelled tasks
  • plan_schedule — Plans multiple work blocks at once
  • update_timetable_entry — Adjusts existing blocks
  • get_free_time_windows — Finds available time slots for rescheduling

Schedule Management

  • Dynamic rescheduling: If a task finishes early, Genesis can move remaining work to free time windows
  • Overlap detection: Prevents scheduling conflicts between tasks
  • Priority handling: High-priority tasks get scheduled first

Free Time Windows

When Work Mode has completed its assigned work, it checks for free time windows using get_free_time_windows(). If additional productive time is available, it may take on related sub-tasks. If no free time or work remains, it signals completion.

Work Mode Status Indicators

Thinking (Cyan): Analyzing schedule and planning work
Executing (Yellow): Actively working on scheduled tasks
Resting (Orange): On a scheduled break
Idle (Gray): Between work blocks or when all work is complete

Working with the Schedule in Work Mode

Work Mode automatically creates timetable entries for active work sessions, break/rest periods, task-specific time allocations, and review/reporting blocks.

Tips for Work Mode

  • Plan your schedule — set realistic time blocks before starting
  • Use specific goals — "Complete the authentication module" works better than "work on security"
  • Include buffer time — schedule extra time for unexpected complexity
  • Monitor progress — check the tracker panel during work sessions
  • Review completed work — after each block, review what was accomplished

Troubleshooting Work Mode

Work blocks not starting: Verify tasks are scheduled in the timetable, check that the goal is set before pressing Play, ensure Genesis state is "idle".

Tasks appearing out of order: Tasks should sort chronologically — check if times are set correctly. Use plan_schedule to reorder existing tasks.

Work mode not stopping: Use the ⏹ Stop Work Mode button in the tracker panel, or use ⏸ STOP / ⏹ STOP ALL from the main controls.
