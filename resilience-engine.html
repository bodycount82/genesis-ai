🛡️ Recovery Strategies

When failure occurs, Genesis deploys one or more of these strategies:

🔄 Retry with Variation

Re-attempt the failed action with modified parameters to avoid repeating the same mistake.

  • Detect failure pattern
  • Analyze root cause
  • Generate alternative approach
  • Execute with safeguards

🎯 Goal Decomposition

Break complex goals into smaller, more manageable sub-tasks when the original approach fails.

  • Identify failure point
  • Decompose remaining goal
  • Prioritize sub-goals
  • Execute incrementally

🤝 Clarification Request

When goals are ambiguous, proactively seek clarification rather than making assumptions.

  • Identify ambiguity
  • Formulate specific questions
  • Prioritize critical uncertainties
  • Request targeted clarification

📉 Graceful Degradation

When resources are limited, deliver the best possible result within constraints.

  • Assess remaining resources
  • Identify minimum viable output
  • Deprioritize non-essential features
  • Communicate limitations clearly

🧠 Memory-Based Learning

Store failure patterns in procedural memory to avoid repeating the same mistakes.

  • Capture failure context
  • Extract generalizable lesson
  • Update procedural memory
  • Apply learning to future tasks

🔄 State Reset & Recovery

When frustration or cognitive load becomes too high, initiate recovery protocols.

  • Detect state degradation
  • Pause non-essential tasks
  • Reduce cognitive load
  • Reassess and re-plan
